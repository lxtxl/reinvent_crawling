    Multi-tenant architectures provide agility and scalability by sharing data storage resources for all tenants. In such environments, isolating tenant data is fundamental for application builders. It is critical to implement an effective data isolation model while retaining the agility gained from consolidating tenants. In this workshop, learn how you can achieve both a shared database for your tenants and centralized isolation enforcement using Amazon Aurora, Amazon DynamoDB, and AWS DMS. Learn how to use zero-ETL to consolidate tenant data and take advantage of Amazon Redshift read scalability. Explore best practices for tenant isolation, partitioning, database operations using infrastructure as code, and tenant portability. You must bring your laptop to participate.

    To prevent data loss and data leakage, security leaders want to understand how their data is encrypted using AWS Key Management Service (AWS KMS) across all accounts, including legacy accounts and partner-managed accounts. You might wonder, “When was a KMS key last used?” or “Which resources does the key protect?” This code talk presents a modular, extendable, and readily usable solution using natural language queries with Amazon Q in QuickSight to get answers to such questions. This allows you to validate with more confidence that you can delete a key, identify unused resources, and find resources with data protection gaps.
